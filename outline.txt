### Ringkasan Proyek: FINSIGHT

**1. Gambaran Umum Proyek**

FINSIGHT adalah aplikasi web berbasis *predictive analytics* yang dirancang untuk startup dan bisnis. Aplikasi ini berfungsi sebagai dasbor interaktif yang memungkinkan pengguna memprediksi metrik bisnis utama berdasarkan data input. Tujuannya adalah untuk mendemokratisasi akses terhadap AI, memberikan alat prediksi yang kuat kepada perusahaan sebagai ganti data transaksional mereka untuk melatih model lebih lanjut. Aplikasi ini memiliki citra startup teknologi yang modern, bersih, dan profesional.

**2. Fungsionalitas Inti**

*   **Prediksi Profit:** Memprediksi jumlah profit yang akan dihasilkan berdasarkan serangkaian parameter (misalnya, produk, promosi, lokasi).
*   **Prediksi Kuantitas:** Memprediksi jumlah unit produk yang akan terjual.
*   **Klasifikasi Profit Tinggi:** Mengklasifikasikan apakah sebuah skenario penjualan berpotensi menghasilkan profit tinggi atau tidak.
*   **Dasbor Terintegrasi:** Menyediakan satu halaman di mana pengguna dapat menjalankan ketiga model prediksi secara bersamaan.
*   **Proposal Kemitraan:** Menawarkan halaman khusus yang menjelaskan model bisnis startup: akses gratis ke alat AI selama 2 tahun dengan imbalan data penjualan dari perusahaan mitra.

**3. Tumpukan Teknologi (Technology Stack)**

*   **Backend:** Python
*   **Web Framework:** Flask
*   **Manipulasi Data & ML:** Pandas, Scikit-learn
*   **Frontend:** HTML5, CSS3, JavaScript
*   **CSS Framework:** Bootstrap 5
*   **Font:** Google Fonts (Poppins)

**4. Struktur File dan Penjelasannya**

```
/FINSIGHT/
├── app.py
├── company_data.csv
├── requirements.txt
├── models/
│   ├── model_high_profit_classifier.pkl
│   ├── model_profit_predictor.pkl
│   └── model_quantity_predictor.pkl
├── static/
│   ├── images/
│   │   ├── logo.png
│   │   └── homepage_illustration.png
│   └── style.css
└── templates/
    ├── base.html
    ├── index.html
    ├── collaboration.html
    ├── classification_page.html
    ├── profit_page.html
    ├── quantity.html
    └── overall.html
```

*   **`app.py`**: Inti dari aplikasi Flask. File ini menangani *routing*, memuat model-model Machine Learning (`.pkl`), dan menyediakan API untuk prediksi. Logikanya telah diperbarui untuk mengirimkan hanya data *dropdown* yang relevan ke setiap halaman model, sesuai dengan fitur yang dibutuhkan oleh model tersebut.
*   **`requirements.txt`**: Mendaftar semua dependensi Python yang dibutuhkan proyek (Flask, Pandas, Scikit-learn).
*   **`company_data.csv`**: Dataset yang digunakan untuk mengisi opsi-opsi *dropdown* pada form input di halaman web.
*   **`models/`**: Direktori yang berisi model-model machine learning yang sudah dilatih dan disimpan sebagai file `.pkl`.
*   **`templates/`**: Berisi file-file HTML.
    *   `base.html`: Template induk yang berisi struktur dasar (navbar, footer, import CSS/JS).
    *   `index.html`: Halaman utama (Homepage).
    *   `collaboration.html`: Halaman yang menjelaskan proposal kemitraan.
    *   `profit_page.html`, `classification_page.html`: Halaman untuk model Profit dan Klasifikasi. Form input di halaman ini sekarang dibuat secara *hardcoded* untuk menampilkan *dropdown* yang spesifik sesuai kebutuhan model masing-masing.
    *   `quantity.html`: Halaman untuk model Kuantitas. Form inputnya sekarang mencakup input numerik (`Price`, `Unit Cost`) dan *dropdown* spesifik.
    *   `overall.html`: Halaman dasbor terintegrasi. Form di sini adalah gabungan dari semua input yang diperlukan oleh ketiga model, termasuk input numerik dan semua *dropdown* kategorikal.
*   **`static/`**: Berisi aset-aset frontend (CSS dan gambar). Tidak ada file JavaScript eksternal yang digunakan.

**5. Alur Kerja Aplikasi & API**

1.  **Inisialisasi:** Saat `app.py` dijalankan, server Flask dimulai. Semua model ML dimuat ke memori, dan opsi *dropdown* yang relevan dari `company_data.csv` juga dimuat.
2.  **Interaksi Pengguna:**
    *   Pengguna membuka aplikasi dan melihat `index.html`.
    *   Pengguna menavigasi ke salah satu halaman model (misal, `/profit`).
    *   `app.py` mem-filter opsi *dropdown* dan hanya mengirimkan yang relevan untuk model profit ke `profit_page.html`.
    *   Pengguna mengisi form yang sekarang sudah spesifik untuk model tersebut.
3.  **Proses Prediksi (Frontend):**
    *   Ketika tombol "Predict" ditekan, JavaScript di dalam file HTML mengambil data dari semua input form (termasuk input numerik jika ada).
    *   JavaScript mengirimkan data ini dalam format JSON ke endpoint API yang sesuai.
4.  **Proses Prediksi (Backend):**
    *   Endpoint API di `app.py` (misal, `/predict/profit`) menerima data JSON.
    *   Data tersebut diubah menjadi DataFrame Pandas.
    *   Model ML yang sesuai digunakan untuk membuat prediksi.
    *   Hasil prediksi dikirim kembali ke frontend sebagai respons JSON.
5.  **Menampilkan Hasil:**
    *   JavaScript di frontend menerima respons JSON dan menampilkan hasilnya secara dinamis di halaman.

**API Endpoints:**
*   `POST /predict/profit`: Menerima data, mengembalikan prediksi profit.
*   `POST /predict/quantity`: Menerima data, mengembalikan prediksi kuantitas.
*   `POST /predict/classification`: Menerima data, mengembalikan klasifikasi profit tinggi (1 atau 0).
